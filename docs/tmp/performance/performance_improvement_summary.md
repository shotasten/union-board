## 🎯 主要課題

### 問題: N+1クエリ問題

**現状**: 12個のイベントに対して、**12回のAPI呼び出し**を実行  
**結果**: ネットワーク往復時間 × 12回 = 6-8秒のロス

**例え**: 
- 現状 = 書類を1枚ずつ12回コピー機に取りに行く
- 改善後 = 書類12枚をまとめて1回で取りに行く

---

## 💡 改善案の概要

### Phase 1（優先度：最高）

**実装内容**: 全データを1回のAPI呼び出しで取得するバッチAPIを追加

**効果**:
- 初回読み込み: 13-14秒 → **3-4秒**（約75%改善）
- API呼び出し: 14回 → **2回**（約85%削減）

**工数**: 約3時間  
**技術難易度**: 低  
**リスク**: 低

---

### Phase 2-3（優先度：高）

**実装内容**: 
- 初回読み込み時に全データを取得し、キャッシュ
- モーダル表示時はキャッシュを使用（API呼び出しなし）

**効果**:
- 初回読み込み: 3-4秒 → **2秒以下**（さらに50%改善）
- モーダル表示: 6秒 → **0.1秒以下**（約98%改善）
- API呼び出し: 2回 → **1回**

**工数**: 約2時間  
**技術難易度**: 低  
**リスク**: 低
