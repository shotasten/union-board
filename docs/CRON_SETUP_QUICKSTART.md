# 定期同期（cron）設定クイックスタート

## 📋 概要

出欠保存時のカレンダー同期をスキップしたため、カレンダーへの反映は**定期同期（cron）**で自動実行する必要があります。

**実装状況**: ✅ `scheduledSync()`関数は既に実装済み  
**設定状況**: ⚠️ トリガーの設定が必要

---

## 🚀 設定手順（5分で完了）

### 1. Google Apps Script エディタを開く

1. スプレッドシートを開く
2. **拡張機能** → **Apps Script** をクリック

### 2. トリガーを追加（2つのトリガーを設定）

#### トリガー1: 通常同期（低頻度）

1. 左側メニューから **「トリガー」**（時計アイコン）をクリック
2. 右下の **「+ トリガーを追加」** をクリック
3. 以下のように設定：

| 項目 | 設定値 |
|------|--------|
| **実行する関数** | `scheduledSync` |
| **イベントのソース** | `時間主導型` |
| **時間ベースのトリガー** | `1時間おき`（推奨） |
| **エラー通知設定** | `即座に通知を受け取る`（推奨） |

4. **「保存」** をクリック

#### トリガー2: 高頻度同期（土日13時前後20分間用）

1. 再度 **「+ トリガーを追加」** をクリック
2. 以下のように設定：

| 項目 | 設定値 |
|------|--------|
| **実行する関数** | `scheduledSyncHighFrequency` |
| **イベントのソース** | `時間主導型` |
| **時間ベースのトリガー** | `5分おき` |
| **エラー通知設定** | `即座に通知を受け取る`（推奨） |

3. **「保存」** をクリック

### 3. 権限の承認

初回実行時に権限の承認が求められる場合があります：
- 「権限を確認」をクリック
- Googleアカウントを選択
- 「詳細」→「[プロジェクト名]に移動」をクリック
- 「許可」をクリック

---

## ✅ 設定完了の確認

### 方法1: トリガー一覧で確認

1. GASエディタの「トリガー」タブを開く
2. 以下の2つのトリガーが表示されていることを確認：
   - `scheduledSync`（1時間おき）
   - `scheduledSyncHighFrequency`（5分おき）
3. 両方の状態が「有効」になっていることを確認

### 方法2: 実行ログで確認

1. GASエディタの「実行数」タブを開く
2. 通常時（土日13時前後以外）：
   - `scheduledSync` が1時間おきに実行される
   - ログに `⏭️ 通常同期スキップ（高頻度同期時間外）` が表示される
3. 土日13時前後20分間（12:40-13:20）：
   - `scheduledSyncHighFrequency` が5分おきに実行される
   - ログに `🚀 高頻度同期実行` が表示される
   - `✅ 定期同期完了: 成功 X件, 失敗 0件` が表示されれば成功

## 📊 動作確認

### 出欠保存の動作

1. **出欠を保存** → 即座に完了（2-3秒）
2. **カレンダー反映**：
   - **通常時**: 最大1時間後に自動反映（通常同期で）
   - **土日13時前後20分間**: 最大5分後に自動反映（高頻度同期で）

### 手動同期（即座に反映したい場合）

1. アプリにアクセス
2. 管理者ログイン
3. ヘッダーの **「🔄」（同期）ボタン** をクリック
4. 即座にカレンダーに反映されます

## ⚙️ 推奨設定

### トリガー頻度

#### 通常同期（`scheduledSync`）

| 頻度 | メリット | デメリット | 推奨度 | 制限使用率 |
|------|---------|----------|--------|-----------|
| **1時間おき** ⭐ | API呼び出しが少ない、最も安全 | 反映が遅い | ⭐⭐⭐ | 約4% |
| **30分おき** | バランスが良い、安全 | API呼び出しがやや多い | ⭐⭐ | 約7% |
| **15分おき** | 反映が速い | API呼び出しが多い、実行時間制限に注意 | ⭐ | 約15% |

**推奨: 1時間おき**
- 通常時は低頻度で十分
- API呼び出し回数を最小化（1日24回）
- 土日13時前後は高頻度同期でカバー
- **最も安全**（制限の約4%のみ使用）

**⚠️ 15分おきについて**:
- 制限内ではあるが、イベント数が多い場合に実行時間制限（6分）のリスクあり
- 詳細は `docs/CRON_FREQUENCY_ANALYSIS.md` を参照

#### 高頻度同期（`scheduledSyncHighFrequency`）

| 頻度 | メリット | デメリット | 推奨度 |
|------|---------|----------|--------|
| **5分おき** ⭐ | 練習前の出欠変更に即座に対応 | 土日13時前後のみ実行 | ⭐⭐⭐ |
| **10分おき** | API呼び出しが少ない | 反映がやや遅い | ⭐⭐ |

**推奨: 5分おき**
- 土日13時前後20分間のみ実行（12:40-13:20）
- 練習前の出欠変更に即座に対応
- 実行回数は限定的（週2回 × 4回 = 8回/週）

### 動作の仕組み

1. **通常時（土日13時前後以外）**:
   - `scheduledSync`（1時間おき）が実行される
   - 関数内で時間帯をチェックし、高頻度時間帯でない場合はスキップ
   - 実際の同期は実行されない（高頻度同期に任せる）

2. **土日13時前後20分間（12:40-13:20）**:
   - `scheduledSyncHighFrequency`（5分おき）が実行される
   - 強制的に同期を実行
   - 練習前の出欠変更に即座に対応

## 🔧 トラブルシューティング

### トリガーが実行されない

**原因:**
- トリガーが正しく設定されていない
- 関数名が間違っている（`scheduledSync` であることを確認）

**対処法:**
1. トリガー設定を確認（関数名: `scheduledSync`）
2. トリガーを削除して再度追加
3. 実行ログでエラーを確認

### カレンダーに反映されない

**原因:**
- トリガーが停止している
- 表示期間外のイベントを編集している

**対処法:**
1. トリガーが有効になっているか確認
2. 編集したイベントが表示期間内か確認
3. 手動で同期ボタンを押して確認

## 📝 補足

### 表示期間の設定

定期同期は**表示期間（DISPLAY_START_DATE ～ DISPLAY_END_DATE）**のイベントのみを同期します。

**設定方法:**
1. アプリにアクセス
2. 管理者ログイン
3. 設定パネル → 表示期間を設定
4. `DISPLAY_START_DATE`: 例 `2025-04-01`
5. `DISPLAY_END_DATE`: 例 `2026-03-31`

表示期間が未設定の場合、デフォルト期間（30日前～1年後）が適用されます。
